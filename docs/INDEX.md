## Project structure (documentation-relevant)

```
WintEHR/
├── README.md                # Project overview + quick start
├── CONTRIBUTING.md          # Contributing guidelines
├── RELEASE_NOTES.md         # Historical release notes
├── AUTOMATED_DEPLOYMENT.md  # Deployment automation notes
├── CLAUDE.md                # Agent context (committed; root of the CLAUDE.md tree)
├── docs/                    # Human-facing reference docs (this directory)
├── backend/                 # FastAPI backend (see backend/CLAUDE.md)
├── frontend/                # React frontend (see frontend/CLAUDE.md)
├── deploy/                  # Deployment configs (HAPI overlay, nginx, NSG script)
└── docker-compose.yml       # Single compose file, dev/prod profiles
```

Notes:
- **`CLAUDE.md` files are committed** and serve as agent/developer context —
  the root file owns project-wide rules; module files carry deltas. Start at
  [`/CLAUDE.md`](../CLAUDE.md).
- There is **one** `docker-compose.yml` with `dev` / `prod` profiles — there is
  no separate `docker-compose.prod.yml`.
- `claudedocs/` holds agent-generated working documents and is gitignored.

### Deploying WintEHR
1. [Deployment Guide](DEPLOYMENT.md) for your target environment
2. [Configuration Guide](CONFIGURATION.md) for config files
3. Azure: [Azure Deployment](AZURE_DEPLOYMENT.md); client VPC: [Client Deployment Playbook](CLIENT_DEPLOYMENT.md)

### Verifying workflows
1. [Testing Checklist](TESTING_CHECKLIST.md) — the reviewer checklist
2. `cd backend && pytest tests/ -v` and `cd frontend && npm test -- --watchAll=false`

## Essential commands

```bash
# Deployment
./deploy.sh                    # Full deployment (dev profile)
./deploy.sh --skip-build       # Skip Docker rebuild
./deploy.sh --environment prod # Prod profile
./deploy.sh status             # Check system status

# Backend
cd backend && pytest tests/ -v

# Frontend
cd frontend && npm start
cd frontend && npm test -- --watchAll=false
cd frontend && npm run lint

# Database / FHIR
docker exec -it emr-postgres psql -U emr_user -d emr_db
curl http://localhost:8888/fhir/metadata
curl 'http://localhost:8888/fhir/Patient?_summary=count'
```

## Contributing to documentation

When code changes documented behavior, update the affected doc in the same
change (see [`docs/CLAUDE.md`](CLAUDE.md) for which doc owns what):

- New deployment option → [DEPLOYMENT.md](DEPLOYMENT.md)
- New configuration → [CONFIGURATION.md](CONFIGURATION.md), `config.example.yaml`
- Architectural change → [SYSTEM_ARCHITECTURE.md](SYSTEM_ARCHITECTURE.md) + root/module `CLAUDE.md`
- New feature → [README.md](../README.md)
- API endpoints → an `API_UPDATES_*.md` entry + nearest `CLAUDE.md`
