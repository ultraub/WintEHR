# docs/ — documentation directory

Human-facing reference documentation for WintEHR (deployment, configuration,
architecture, CDS Studio, security). **Inherits** root `CLAUDE.md`.

This is NOT where agent-generated analysis goes — those reports belong in
`claudedocs/`. This directory is durable reference prose for developers; agent
context lives in `CLAUDE.md` files next to the code they describe.

---

## Conventions

- File names: `UPPER_SNAKE_CASE.md`.
- `architecture.png` is generated by `generate_architecture.py` — regenerate it
  rather than hand-editing the image.
- Reference prose for humans goes here. Agent-facing architecture/pattern context
  goes in the nearest `CLAUDE.md`. Keep the two separate — don't put tutorial
  prose in a `CLAUDE.md`, don't put agent rules here.

## When to update which doc

| Change | Update |
|---|---|
| Deployment / config behavior | `DEPLOYMENT.md`, `CONFIGURATION.md` (+ `deploy/CLAUDE.md`) |
| CDS Studio / visual builder | `CDS_STUDIO_*.md` |
| CQL authoring surface | `STUDENT_CQL_PRIMER.md` (+ `backend/api/cds_hooks/CLAUDE.md`) |
| Architecture pattern | `SYSTEM_ARCHITECTURE.md` + root & module `CLAUDE.md` |
| API endpoints | `API_UPDATES_*.md` + nearest `CLAUDE.md` |

When code changes documented behavior, update the affected doc in the same change —
a stale reference doc misleads the next reader the same way a stale `CLAUDE.md` does.
